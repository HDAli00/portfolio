Ch. 1 Field notes

Principles with scars behind them.

One card per lesson, in the order production taught them. Flip through — or let the deck riffle on its own.

Observability

Alert on symptoms, never on causes.

Cause-based alerts rot the moment the architecture shifts. Page on what the user feels; let dashboards explain why.

note 014 — learned at 03:00, CET
Kubernetes

If a human has to SSH in, the platform has failed.

Every manual intervention is a missing abstraction. Count them. Then delete them, one by one.

note 011 — running tally: 3 left
Pipelines

Boring pipelines are a feature, not a gap.

Clever CI is a liability at 2 a.m. The best deploy is the one nobody has to think about.

note 009 — est. 2023, still true
On-call

Runbooks are apologies written in advance.

If the runbook has more than five steps, the fix belongs in code. Write the apology once, then automate it away.

note 007 — after incident #38
Craft

Quiet infrastructure is loud work.

Nobody notices the platform when it's good. That silence is the deliverable — budget for it.
